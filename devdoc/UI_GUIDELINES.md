# Remission UI Guidelines (Apple OS 26.0+ Liquid Glass Era)

Цель: полное использование возможностей SDK 26.0+ для создания иммерсивного, динамичного и нативного интерфейса с использованием концепции **Liquid Glass**.

## Принципы Liquid Glass

- **Динамическое стекло**: Вместо статических материалов (`Materials`) используем `Glass`. Оно отражает свет, реагирует на окружение и взаимодействия пользователя в реальном времени.
- **Морфинг и контейнеры**: Используем `GlassEffectContainer` для объединения связанных элементов управления в единые «живые» формы.
- **Материализация**: Переходы и появление элементов должны использовать `GlassEffectTransition.materialize` для создания ощущения физического проявления интерфейса.

## Токены и код-стандарты (v26)

Стандарты инкапсулированы в:
- `Remission/Views/Shared/AppTheme.swift` — токены Liquid Glass.
- `Remission/Views/Shared/AppChrome.swift` — модификаторы поверхностей нового поколения.

### Основные модификаторы:

- `View.appCardSurface(cornerRadius:)` — применяет `glassEffect(.regular)` к скругленному прямоугольнику.
- `View.appPillSurface()` — применяет `glassEffect` к капсульной форме.
- `View.appMaterialize()` — анимированный переход материализации стекла.

## Поверхности (Surfaces)

### Карточки и Панели
- **Эффект**: `glassEffect(.regular)`.
- **Скругление**: `14pt` (continuous).
- **Контейнеры**: Большие блоки информации (списки серверов, детали) группируются в стеклянные поверхности.

### Интерактивные элементы (Pills)
- **Форма**: `Capsule()`.
- **Контейнер**: Группы кнопок или поисковые строки всегда оборачиваются в `GlassEffectContainer`. Это позволяет системе анимировать «слияние» или «разделение» элементов управления при навигации.

## Цвет и типографика (Dynamic Vibrancy)

- **Vibrancy**: Текст `.foregroundStyle(.primary)` и `.secondary` внутри стеклянных контейнеров автоматически приобретает динамическую яркость, адаптируясь под цвета фона.
- **Accent**: Использование `Color.accentColor` обязательно. Система 26.0+ активно использует акцентный цвет для «подсветки» краев стеклянных поверхностей.

## Do / Don’t

**Do:**
- Использовать `GlassEffectContainer` для всех групп кнопок в тулбарах и списках.
- Применять `.appMaterialize()` для появления всплывающих или динамических панелей.
- Полагаться на нативную форму `Capsule` для поисковых полей.

**Don’t:**
- Использовать старый `.background(.regularMaterial)`. В SDK 26+ он считается устаревшим в пользу `glassEffect()`.
- Задавать непрозрачные фоны, которые блокируют отражения и преломления Liquid Glass.
